# VirtualRoom package allows you to build tousands of rooms very similar each other
# using few configuration files.
#
# You can build a grid of rooms or places without specifically programming each room.
# In the grid each room is identified on a map by a couple of coordinates (x horizontal,
# y vertical); the grid is based on few configuration file: 
# ccc_grid.txt, ccc_prop.txt, ccc_short.txt, ccc_long.txt, 
# where "ccc" shoul be an acronym that identifies the map itself. 
# In this way it is possible to define several maps.
# 
# File ccc_clon.txt contains descriptions for cloned-objects (like add_object).
# File ccc_center.txt contains a list of main towns within the grid.
# 
# In the file ccc_grid.txt the map logic-wideness is given by the length of the first 
# usable row (divided by 2 and minus 2), because each cell is defined by a couple of 
# characters. The map logic-height is given by the number of usable rows.
# 
# In particular, codes that begins with "0" are "special" and "inaccessible" and the 
# adjacent cells give information of that with a small standard wording.
# 
# First colums is not part of the grid, so it can be used as ruler. Last column also.
#
# In our case, the Mondo Emerso map is defined by a file "em_grid.txt" which defines
# a 124 x 100 cells grid. Definitions of the "couple of characters" are coded in the
# files em_short.txt, em_long.txt, em_clon.txt
# Righe che iniziano con # vengono saltate.
#
# La "larghezza logica" della mappa č data dalla lunghezza della prima riga utile / 2 -2.
# L' "altezza" logica č espressa dal numero di righe utili.
# In questo caso, il Mondo Emerso č stato suddiviso in una griglia 124 x 100 (=12400).
# Ogni "cella" č formata da due bytes che vanno codificati nei file em_short e em_long .txt
# I codici che iniziano con "0" sono "speciali", non accessibili: le celle adiacenti 
# ne danno notizia dicendo "nelle vicinanze: ovest, Fiume Saar..."
# La coordinata x č in orizontale, y in verticale. 
# I numeri del "righello" individuano quadratini 4x4 della mappa cartacea che ho usato
# durante la conversione da carta a file.
# La prima colonna non fa parte della mappa e puň essere usata come righello.
# Anche l'ultima colonna non fa parte della mappa e puň essere usata come righello.
# I wizards usano "reload" per ricaricare in runtime tutti i file.
